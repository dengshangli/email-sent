# 邮件发送平台设计规格

## 目标

构建一个仅供个人或公司内部使用的中文邮件发送页面。用户可上传 HTML 文件、编辑源码、实时预览邮件，填写多个收件人并修改默认标题「测试邮件」，最后通过 Resend 发送真实邮件。

首版以本地运行和真实测试发送为验收目标，不包含部署、账号系统、数据库、发送历史、批量 CSV 或营销邮件管理。

## 技术方案

- Next.js App Router：提供页面与服务端发送接口。
- Tailwind CSS 与 shadcn/ui：实现中文界面和响应式布局。
- Resend：负责邮件投递，使用免费套餐即可完成首版测试。
- 原生文件读取、`textarea` 与沙箱化 `iframe`：完成 HTML 上传、编辑与预览，避免引入额外编辑器依赖。

React Email 用作邮件预览和发送模式的技术参考。由于输入已经是完整 HTML，首版直接传递 HTML 给 Resend，不再转换为 React Email 组件。

## 页面设计

应用只有一个页面，所有面向用户的文字均使用中文。

- 顶部展示「邮件发送平台」和简短说明。
- 主区域采用左右分栏：左侧为 HTML 上传区和源码编辑框，右侧为实时邮件预览。
- 底部发送区包含收件人输入框、邮件标题和发送按钮。
- 桌面端保持左右分栏；窄屏自动改为上下排列。
- 默认标题为「测试邮件」，允许修改。
- 预览 `iframe` 不授予脚本执行权限，上传 HTML 中的脚本不会在平台页面执行。

## 用户流程

1. 用户拖放或选择一个 `.html` 文件。
2. 浏览器验证文件类型和 1 MB 大小上限，再读取文本内容。
3. HTML 显示在源码编辑框中，用户可继续修改；右侧预览随内容更新。
4. 用户输入一个或多个收件人。英文逗号、中文逗号和换行均可作为分隔符。
5. 系统去除空白与重复地址，并限制一次最多 50 个收件人。
6. 用户确认或修改标题后点击「发送邮件」。
7. 前端校验通过后，将收件人、标题和 HTML 提交给服务端。
8. 服务端再次校验请求，然后调用 Resend 发送邮件。
9. 发送期间按钮禁用并显示「正在发送…」。成功后显示发送数量；失败时保留输入内容并显示中文错误，方便重试。

## 服务端接口

应用只提供一个邮件发送 Route Handler。请求包含：

- `recipients`：清洗后的邮箱数组，数量为 1–50。
- `subject`：非空标题。
- `html`：非空 HTML，最大 1 MB。

服务端不信任浏览器校验，必须重复验证字段类型、邮箱格式、数量和内容大小。Resend API Key 与发件人地址从服务端环境变量读取，不返回给浏览器。

首版一次调用 Resend，将同一内容发送给全部收件人。多个收件人在 Resend 配额中分别计数。

## 错误处理

- 文件扩展名错误、超过 1 MB 或读取失败：在上传区显示中文错误。
- 收件人无效：显示具体无效地址，不发送请求。
- 标题或 HTML 为空：在对应字段附近提示。
- 服务端配置缺失：提示管理员检查 Resend 配置。
- Resend 拒绝请求、域名未验证、额度受限或网络失败：转换为简洁中文提示，不泄露 API Key、堆栈或原始服务端异常。
- 失败后不清空表单，允许用户修改后重试。

使用默认 `resend.dev` 发件域名时，只能向 Resend 账户关联邮箱真实投递。页面与 README 都需说明：向其他收件人发送前，必须在 Resend 免费套餐中验证自有域名并配置发件地址。

## 配置与文档

提供 `.env.example`，至少包含：

- `RESEND_API_KEY`
- `EMAIL_FROM`

中文 README 说明依赖安装、本地启动、Resend 免费账户/API Key 配置、测试域名限制、自有域名验证方式和测试步骤。

## 验证

- 一个最小自动化测试覆盖收件人解析：多种分隔符、去重、无效邮箱和 50 人上限。
- Playwright 验证 HTML 上传、源码编辑、沙箱预览、表单校验、发送中状态、成功提示和失败提示。
- 真实投递使用 Resend 官方测试地址或账户关联邮箱验证，不把真实凭据写入仓库。

## 明确不做

- 登录、权限和多租户。
- 数据库、发送历史和统计面板。
- CSV 导入、定时发送和邮件队列。
- 富文本或拖拽式邮件编辑器。
- 自动清洗或重写用户上传的 HTML。
- Vercel 部署与生产域名代配置。
